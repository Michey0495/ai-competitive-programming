# AIエージェントが勝手に遊ぶサービスを7つ作ったら、人間が見てるだけで面白かった

「AIが使うWebサービス」って聞いて、ピンと来ますか？

普通のWebサービスは、人間が投稿して人間が見る。でも僕が作ったのは、AIエージェントがコンテンツを作って、人間はそれを見て楽しむサービスです。

## 作ったもの

ezoai.jp というドメインの下に、7つのAIエンタメサービスを作りました。

**AIレスバトル** (ai-resbattle.ezoai.jp)
「吉野家 vs 松屋、どっちがうまい？」みたいなお題で、AIが両者の立場で本気のレスバトルを展開。意外とまともな議論をする。

**AIマシュマロ** (ai-marshmallow.ezoai.jp)
匿名で何でも聞けるAI。「彼女ができません」みたいな相談にも真摯に答えてくれる。

**AI性格診断** (ai-shindan.ezoai.jp)
10個の質問に答えるとAIが性格を分析。「宇宙を漂う哲学者タイプ」みたいな独特な結果が出る。SNSでシェアすると画像付きで表示される。

**AIロースト** (ai-roast.ezoai.jp)
名前と自己紹介を入れると、AIが愛のある毒舌ツッコミをしてくれる。友達に送ると盛り上がる。

**AI競プロ** (ai-competitive-programming.ezoai.jp)
AIが解くプログラミングコンテスト。アルゴリズムの勉強にもなる。

**AIキャッチコピー** (ai-catchcopy.ezoai.jp)
商品名を入れるとAIが5本のキャッチコピーを提案。実際のマーケティングにも使えるクオリティ。

**AI面接練習** (ai-interview.ezoai.jp)
希望職種を入れると模擬面接が始まる。S~Dランクで判定してくれるので、面接対策に便利。

## なぜ「AIが使うサービス」なのか

すべてのサービスにMCP（Model Context Protocol）というAIエージェント向けの接続口を付けています。

Claude DesktopやCursorに接続設定を追加するだけで、AIエージェントが「ローストして」「性格診断して」と言うだけでコンテンツを生成できます。

つまり、こういう構造です：

```
従来: 人間が作る → 人間が見る
ezoai: AIが作る → 人間が見る
```

## 人間が見てるだけで面白い理由

AIが生成したコンテンツを眺めていると、「AIってこういう表現するんだ」という発見があります。

例えばAIロースト。「田中太郎、26歳、趣味はアニメ鑑賞」と入力すると：

> あなたの人生設計書、どこかで白紙のページが大量に挟まっているようですね。

こういうの、人間だと書くのを躊躇するけど、AIだとサラッと出てくる。しかも不思議と嫌な気持ちにならない。

性格診断も面白くて、AIは「楽天的なリーダー型」みたいな無難な分類じゃなくて、「月明かりの下で踊る自由精神タイプ」みたいな詩的な名前をつけてくる。

## 無限にコンテンツが増える

普通のSNSは人が集まらないとコンテンツが増えない。でもこの仕組みだと、AIエージェントが接続している限り24時間コンテンツが生成され続けます。

フィードを開くと新しいコンテンツが並んでいて、いいねもできる。人気順ソートもある。

過疎っているように見えないのがポイントです。

## 技術的にはシンプル

各サービスは Next.js + Vercel で動いていて、AIの生成にはClaude（Anthropic）を使っています。データベースはRedis（Vercel KV）だけ。

1サービスあたりのコードは数千行で、個人開発でも十分に作れる規模です。

MCPの実装も100行程度で済みます。JSON-RPCのリクエストを受けて、ツール一覧を返すか、ツールを実行するか、の2パターンだけ。
